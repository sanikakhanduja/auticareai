## Language Persistence
- User's selected language is saved in localStorage
- Persists across page refreshes and sessions
- Defaults to English if no preference is set

## Usage Pattern for Developers

To add translations to a new page or component:

```tsx
import { useTranslation } from "react-i18next";

export default function MyComponent() {
  const { t } = useTranslation();
  
  return (
    <div>
      <h1>{t("mySection.title")}</h1>
      <p>{t("mySection.description")}</p>
    </div>
  );
}
```

## Notes

- The system uses i18next with a browser language detector plugin
- Falls back to English if browser language is not Hindi
- All translation keys follow a hierarchical naming convention (e.g., "screeningTest.eyeContact", "portal.addChildProfile")
- Both translation files have identical key structures for consistency
- Memoization used for translated arrays (questions, processing steps) to prevent re-renders
